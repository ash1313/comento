1) HTTP 통신:
   1. HTTP는 Hyper Text Transfer Protocol의 약자이다.
   2. 인터넷에서 데이터를 주고받는 프로토콜을 말한다.
   3. 요청(request)와 응답(response)으로 이루어져 있다.
    3_1. 클라이언트가 서버에 요청을 보냄
    3_2. 서버는 요청에 대한 응답 결과를 줌
    3_3. 클라이언트 사용자에게 응답 받은 결과를 보여줌
2) 브라우저에 URL을 입력 후 요청하여 서버에서 응답하는 과정까지에 대해 학습 한 내용을 작성
   1. 브라우저에 URL을 입력한다.
   2. 브라우저는 url에 상응하는 IP주소를 찾기 위해 DNS기록 캐시를 확인한다.
    2_1. browser -> os -> router -> ISP
   3. 만약에 요청한 URL이 캐시에 없으면 ISP의 DNS서버는 url을 갖는 서버의 IP주소를 찾기 위해 DNS query를 시작한다.
   4. 브라우저는 서버와 TCP 통신을 시작한다.
    4_1 클라이언트 기계는 서버가 새 연결을 위해 열려있는지 물어보며 서버에 SYN 패킷을 보낸다.
    4_2 만약 서버가 새 연결을 시작할 수 있는 오픈 포트를 가진다면, SYN/ACK 패킷을 사용하여 SYN패킷의 ACKnowledgement로 응답한다.
    4_3 클라이언트가 서버로부터 SYN/ACK 패킷을 받을 것이고 서버에 ACK패킷을 보냄으로써 받았음을 알려줄 것이다.
   5. 브라우저는 웹 서버에 HTTP 요청을 보낸다.
   6. 서버는 요청을 처리하고 응답을 다시 보낸다.
   7. 서버는 HTTP 응답을 내보낸다.
   8. 브라우저는 HTML 컨텐츠를 표시한다.
